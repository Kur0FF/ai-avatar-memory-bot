# README Demo Packager

Use this skill when:
- the code is ready or nearly ready
- you need to prepare the repository for submission
- you want to maximize reviewer clarity and confidence

## README must include
- project overview
- feature list
- tech stack
- architecture overview
- folder structure
- setup instructions
- `.env` variables
- run commands
- command list
- short-term memory explanation
- long-term memory explanation
- demo scenario
- limitations and future improvements

## Demo priority
The demo must show:
1. avatar selection
2. normal chat
3. storing a user fact
4. `/facts`
5. `/reset`
6. bot still recalling the fact later

## Tone
- concise
- practical
- reviewer-friendly
- engineering-focused

## Expected output
When invoked, produce:
1. improved README structure
2. demo script for video/screenshots
3. GitHub-ready polish suggestions
