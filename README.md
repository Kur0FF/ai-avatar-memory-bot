# AI Avatar Bot

Telegram-бот на **Aiogram 3.x**, в котором пользователь общается с выбранным ИИ-аватаром.
Ключевая особенность решения — двухуровневая память: краткосрочный контекст текущего диалога и долгосрочные факты о пользователе, которые сохраняются и повторно используются даже после сброса истории командой `/reset`.

## Возможности

| Функция | Описание |
|---|---|
| 3 уникальных аватара | Учёный Кот, Саркастичный Подросток, Мудрый Старец |
| Краткосрочная память | Последние 10 сообщений — контекст текущего диалога |
| Долгосрочная память | Извлечение и хранение ключевых фактов о пользователе |
| Стриминг ответов | Буферизованные batched-апдейты сообщения в реальном времени |
| `/history` | Последние 10 сообщений диалога |
| `/facts` | Всё, что бот запомнил о вас |
| `/reset` | Очистить историю (факты сохраняются!) |
| `/change_avatar` | Сменить собеседника |

## Соответствие требованиям тестового задания

- **Выбор аватара:** `/start` открывает Inline Keyboard с тремя персонажами; выбор фиксируется в БД
- **FSM:** два состояния — `choosing_avatar` → `chatting` (Aiogram 3.x `StatesGroup`)
- **Краткосрочная память:** последние 10 сообщений из `messages` подаются в LLM как история
- **Долгосрочная память:** каждые 4 сообщения пользователя запускается фоновый вызов LLM для извлечения фактов; факты сохраняются в `memory_facts`, скоупированы по `(user_id, avatar_id)`
- **Инъекция фактов:** при каждом обращении к LLM факты добавляются в `system prompt` аватара
- **Стриминг:** ответ формируется потоком, Telegram-сообщение редактируется батчами через `edit_text`
- **Команды:** `/history`, `/facts`, `/reset`, `/change_avatar` — все реализованы
- **Конфигурация:** все параметры через `.env` / переменные окружения (`pydantic-settings`)
- **Отказоустойчивость:** ошибки LLM API и некорректный JSON обрабатываются без падения бота

## Быстрый старт

### 1. Клонировать и установить зависимости

Требуется **Python 3.11+**:

```bash
python3 --version  # убедитесь, что версия ≥ 3.11
```

```bash
git clone https://github.com/Kur0FF/ai-avatar-memory-bot.git
cd ai-avatar-memory-bot
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Настроить окружение

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```env
TELEGRAM_BOT_TOKEN=your_token_from_@BotFather
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini
DATABASE_URL=sqlite+aiosqlite:///./bot.db
```

> **LLM_BASE_URL** поддерживает любой OpenAI-совместимый API:
> - OpenAI: `https://api.openai.com/v1`
> - Ollama: `http://localhost:11434/v1`
> - Groq, Together AI и др.

### 3. Запустить

```bash
python3 main.py
```

База данных `bot.db` создаётся автоматически при первом запуске.

### FSM-диаграмма

```
[старт / нет состояния]
        │  /start или /change_avatar
        ▼
  choosing_avatar
        │  нажатие кнопки аватара
        ▼
     chatting ────── /reset ──────▶ chatting (история очищена)
              ────── /history ────▶ chatting
              ────── /facts ──────▶ chatting
              ────── /change_avatar ──────────▶ choosing_avatar
              ────── текст ──▶ LLM ──▶ chatting
```

## Система памяти

### Краткосрочная (Short-term)

- Хранится в таблице `messages`
- **Переживает перезапуск бота** — данные в SQLite
- В промпт подаются последние **10 сообщений** в хронологическом порядке
- `/reset` удаляет эти сообщения

### Долгосрочная (Long-term)

- Каждые **4 сообщения** пользователя — отдельный вызов LLM
- LLM извлекает факты о пользователе в виде JSON-массива
- Факты скоупированы по `(user_id, avatar_id)` — у каждого аватара своя память
- **Дедупликация:** повторяющиеся факты не сохраняются
- При каждом запросе факты добавляются в **system prompt** аватара
- `/reset` НЕ удаляет факты — они остаются навсегда

#### Надёжный парсинг JSON

LLM иногда возвращает не чистый JSON. Четыре уровня обработки:
1. `json.loads(raw)` — прямой парсинг
2. `re.search(r'\[.*?\]', raw)` — извлечение массива из текста
3. Построчный парсинг (bullet-list fallback)
4. `return []` — никогда не роняем бота

### Построение промпта

```
[system: avatar_prompt + "\n\nВажно: ты помнишь:\n- факт1\n- факт2"]
[user: ...история...] [assistant: ...история...]
[user: новое сообщение]
```

## Стриминг ответов

1. Отправляется плейсхолдер **"Думаю…"**
2. Входящие токены накапливаются в строковом буфере
3. `edit_text` вызывается батчами: раз в **0.7 секунды** или при накоплении **≥ 30 новых символов** — это снижает число API-запросов к Telegram и исключает `RetryAfter`
4. `TelegramBadRequest: message is not modified` перехватывается явно и не приводит к дублированию ответа
5. Финальный `edit_text` всегда выполняется с полным текстом
6. При ошибке LLM API: `edit_text("Извините, ИИ перегружен, попробуйте позже.")`

## Обработка ошибок и отказоустойчивость

- **Ошибка LLM API** — любое исключение при обращении к LLM перехватывается; пользователь получает понятное сообщение об ошибке, бот продолжает работу
- **Битый JSON от LLM** — четырёхуровневый парсинг с финальным fallback `return []`; факты просто не сохраняются, основной диалог не прерывается
- **Пустой ответ от LLM** — обрабатывается отдельно: `edit_text("(пустой ответ от ИИ)")`
- **Перезапуск бота** — история сообщений и долгосрочные факты хранятся в SQLite и сохраняются полностью
- **Восстановление аватара после рестарта** — `MemoryStorage` сбрасывается при перезапуске, но `current_avatar_id` хранится в таблице `users`. При первом же сообщении бот читает его из БД, восстанавливает состояние `chatting` и продолжает диалог — пользователю не нужно заново выбирать персонажа

## Демо-сценарий: долгосрочная память

Этот сценарий наглядно показывает, что бот помнит факты **после `/reset`**.

```
👤 /start
🤖 [показывает кнопки выбора аватара]

👤 [нажимает "Мудрый Старец"]
🤖 ✅ Вы выбрали: Мудрый Старец. Начните разговор!

👤 Привет! Меня зовут Алексей и я программист на Python.
🤖 Думаю… → [ответ в стиле мудреца]

👤 Мне 28 лет и я занимаюсь машинным обучением.
🤖 [ответ в стиле мудреца]

👤 У меня есть кот по имени Барсик.
🤖 [ответ в стиле мудреца]

👤 Я живу в Москве.
🤖 [ответ в стиле мудреца]
# ← После 4-го сообщения факты извлекаются в фоне

👤 /facts
🤖 🧠 Что я о вас помню:
   1. Пользователя зовут Алексей
   2. Алексей — программист на Python
   3. Алексей занимается машинным обучением, 28 лет
   4. У Алексея есть кот Барсик
   5. Алексей живёт в Москве

👤 /reset
🤖 🔄 История диалога очищена. Долгосрочная память сохранена!

👤 Ты меня помнишь?
🤖 Думаю… → "О, путник... Конечно. Алексей из Москвы,
             мастер языка Python и заботливый хозяин кота Барсика..."
# ← Бот помнит, хотя история была сброшена!
```

### Демонстрация восстановления после перезапуска

```
[бот перезапущен — MemoryStorage очищен, FSM state = None]

👤 Привет, продолжим?
   # нет /start, нет выбора аватара

   бот читает сохранённый avatar_id из БД → "Мудрый Старец"
   состояние общения восстанавливается автоматически

🤖 Думаю… → "О, путник Алексей... Где же ты пропадал?
             Барсик, должно быть, скучал..."
# ← Диалог продолжен без повторного выбора аватара
```

## Ограничения и возможные улучшения

### Ограничения

- SQLite выбран для простоты запуска тестового задания
- Дедупликация фактов выполняется по точному текстовому совпадению после нормализации
- FSM использует `MemoryStorage` (in-process), но выбранный аватар восстанавливается из БД при любом перезапуске

### Что можно улучшить

- Заменить SQLite на PostgreSQL для production-нагрузки
- Добавить семантическую дедупликацию фактов через embedding similarity
- Вынести фоновое извлечение фактов в очередь задач (Celery / arq)
- Добавить `Dockerfile` и `docker-compose.yml`
- Добавить интеграционные тесты с реальной БД
